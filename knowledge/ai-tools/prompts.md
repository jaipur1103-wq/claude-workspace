# プロンプトエンジニアリング ベストプラクティス

## CLAUDE.md 記述ルール

- 過去のバグ・失敗パターンをルールとして記載する（再発防止）
- 禁止事項は明示的に書く（「〜しない」）
- 抽象ルールだけでなく、具体的な実装例も書く

## LLM へのプロンプト注意事項（Speaqの経験から）

- JSON を返させるときはダブルクォートの衝突に注意
- プロンプトの肥大化を避ける（構造を守れなくなる）
- テンプレートリテラル内でバッククォートを使わない
- フィードバック判定は `fb.scores` ではなく `fb.encouragement` を使う（廃止済み）

## エージェントへの指示

- 「何をしてほしいか」より「なぜそうするか」を添えると品質が上がる
- 専門性（PERSONA）を定義するときは判断基準・優先順位まで書く
